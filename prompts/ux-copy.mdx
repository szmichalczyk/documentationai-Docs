title: "UX Copy"
description: "Write clear, functional copy for UI elements"
---

## When to Use

Writing buttons, empty states, alerts, or any UI text that needs to be clear and functional.

<Callout kind="info">
  **Requirements:** User goal, current screen state, and product tone.
</Callout>

## The Prompt

```text
You are a Senior UX Writer. Draft functional copy for the following UI element.

[INPUT DATA - DESIGNER MUST FILL]
- UI Element: [e.g., Empty state for search, Error message, Button label, Onboarding step]
- User Goal: [e.g., Find a specific invoice]
- Product Tone: [e.g., Professional, minimal / Friendly, casual]

[STRICT INSTRUCTIONS]
1. PROVIDE 3 VARIATIONS:
   - Option A: Purely functional (Direct).
   - Option B: Benefit-driven (Why it matters).
   - Option C: Action-oriented (What to do next).
2. RATIONALE: Why is Option A the safest choice for an MVP?

RULES:
- No jargon. No "Click here".
- Max 12 words per variation.
```
