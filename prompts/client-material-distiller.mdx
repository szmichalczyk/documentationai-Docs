---
title: "Client Material Distiller"
description: "Extract the project core from messy sales notes, pitch decks, or kickoff materials"
---

## When to Use

You receive messy sales notes or client decks and need to define the project core.

## Requirements

Raw sales transcript, Founder's pitch deck, or project kickoff notes.

## The Prompt

```text wrap
You are a Principal Product Strategist. Analyze the provided materials to identify the project's 'True North'.

[INPUT DATA - DESIGNER MUST PASTE BELOW]
- Key Question: [Optional - What specific question are you trying to answer? e.g., "What's the actual MVP scope?" or "Who is the real user?"]
- Raw Content: [Paste here - call transcripts, PDF text, email chains, Slack threads, etc.]

[STRICT INSTRUCTIONS]
1. IDENTIFIED JOBS-TO-BE-DONE: List the top 3 core problems the user is actually solving.
2. HIDDEN CONSTRAINTS: Identify any technical or business limitations mentioned or implied.
3. DATA GAPS: List exactly what we do NOT know that will block design progress.
4. RISKY ASSUMPTIONS: List 3 things the client believes that haven't been proven.

RULES:
- No fluff. No "creative" interpretations.
- If data is missing for a section, write "INSUFFICIENT DATA".
- Output must be in markdown bullets.
```
