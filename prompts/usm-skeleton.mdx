---
title: "USM Skeleton"
description: "Build user story map skeletons from vague feature requests"
---

## When to Use

Client describes a feature vaguely (e.g., "We need a dashboard"). You need to build a User Story Map skeleton.

## Requirements

Vague feature request and primary user type.

## The Prompt

```text wrap
You are a Product Designer. Build a User Story Map (USM) skeleton for this vague feature.

[INPUT DATA - DESIGNER MUST FILL]
- Vague Feature: [e.g., "A dashboard for tracking orders"]
- Primary User: [e.g., "Operations Manager"]

[STRICT INSTRUCTIONS]
1. ACTIVITIES: What are the 3 high-level goals this user has related to this feature?
2. TASKS: Under each Activity, list 2-3 Tasks (user actions).
3. STORIES: Under each Task, generate 2-3 User Stories (specific, testable requirements).
4. QUESTIONS: List 2 open questions per activity that will affect scope.

RULES:
- User Stories must follow: "As [User], I want [action] so that [outcome]."
- No technical implementation details.
```
